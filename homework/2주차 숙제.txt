서버와 클라이언트란 무엇인가?
1)서버
-네트워크를 통해 클라이언트(사용자 컴퓨터, 스마트폰 등)의 요청을 받아 데이터나 서비스를 제공하는 컴퓨터 또는 프로그램
-24시간 가동되며 웹사이트, 파일, 게임, 메일 등 다양한 리소스를 저장하고 관리하는 역할을 수행
2)클라이언트
-컴퓨터 네트워크 환경에서 서버에 데이터를 요청하고 서비스를 제공받는 주체로, 사용자 컴퓨터, 스마트폰, 혹은 웹 브라우저나 앱 같은 애플리케이션을 의미
-서버와 통신하며 사용자 인터페이스를 담당하고, 서버로부터 받은 정보를 보여주는 역할을 수행

서버와 클라이언트의 관계는 어떻게 되는가?
-네트워크를 통해 데이터를 주고받는 상호의존적 관계 
-클라이언트(사용자 기기)가 서비스나 정보(웹페이지, 데이터 등)를 요청하면, 서버(중앙 컴퓨터)가 이를 처리해 응답하는 1:N 형태의 구조
-클라이언트는 화면 표시와 사용자 입력, 서버는 데이터 저장 및 핵심 로직 처리를 담당

Live Server 로 실행시킬때 나오는 이 주소의 의미가 서버라는 개념과 어떤 관계가 있는가?
http://127.0.0.1:5500/fe_study/html/start.html
현재 사용 중인 컴퓨터가 웹 브라우저(클라이언트)의 요청을 처리하는 '웹 서버'의 역할을 수행하고 있음**을 완벽하게 보여주는 지표

1. 주소
- `http://` (프로토콜)
브라우저(클라이언트)와 서버가 데이터를 주고받기 위한 통신 규칙 
이 주소로 접속했다는 것은, 브라우저가 내 컴퓨터 하드디스크에 있는 파일을 단순히 '읽어온 것'이 아니라, 서버에게 http 규칙에 맞춰 파일을 요청하고 받아왔다는 뜻
-`127.0.0.1` (로컬호스트)
네트워크상에서 내 컴퓨터 자신을 가리키는 전 세계 공통의 약속된 IP 주소
즉, 외부에 있는 다른 서버(예: 구글이나 네이버 서버)에 접속한 것이 아니라, 내 컴퓨터 내부에 띄워진 서버로 찾아갔다는 의미. (`localhost`라는 문자로 대체 가능)
-`:5500` (포트 번호)
하나의 컴퓨터 안에는 수많은 프로그램이 동시에 존재 
포트 번호는 그중에서 '어떤 프로그램(서버)이 이 요청을 처리할 것인가'를 지정하는 출입구 번호 
Live Server 확장 프로그램이 기본적으로 컴퓨터의 5500번 문을 열고 브라우저의 요청을 기다리고 있는 상태
`/fe_study/html/start.html` (경로)
서버가 열려있는 기준 폴더(Root)로부터 해당 파일이 위치한 구체적인 경로 
클라이언트가 "5500번 문에 있는 서버야, 이 경로에 있는 파일 좀 브라우저 화면에 띄워줘"라고 요구하는 구체적인 목적지

2. '서버' 개념과의 본질적인 관계
1. 실제 웹 환경의 완벽한 모방
실제 우리가 인터넷을 사용할 때, 웹 브라우저는 멀리 떨어진 서버 컴퓨터에 접속하여 HTML, CSS, JS 파일을 받아옵니다. Live Server는 내 컴퓨터의 하드디스크 일부를 실제 외부 서버처럼 작동하도록 가상의 환경을 구축

2. 보안 및 기능적 제약 해제 (CORS)
가장 중요한 차이점입니다. 브라우저는 보안상의 이유로 `file:///` (단순 파일 열기) 상태에서는 자바스크립트의 핵심 기능들을 차단

3. 실시간 양방향 통신 (웹소켓)
Live Server는 단순히 파일을 전달하는 것에 그치지 않고, HTML 파일에 몰래 작은 자바스크립트 코드를 주입
이 코드는 에디터(VS Code)와 웹 브라우저 사이에 실시간 연결 통로를 만들어, 코드를 저장할 때마다 서버가 브라우저에게 "파일이 변경되었으니 새로고침 해!"라는 신호를 보냄

Live Server 확장프로그램을 설치해서 사용하고 있는데,
Live Server는 Server 라는 이름을 가지고 있다. Live Server 가 제공해주는 기능은 무엇인가?

-Live Server라는 이름에 걸맞게, 이 확장 프로그램은 단순히 컴퓨터에 있는 파일을 열어주는 것을 넘어 개발자를 위한 '경량화된 가상 웹 서버'를 띄워주는 역할
1. 실시간 새로고침
2. 로컬 http 서버 환경 제공 (보안/기능 제약 해제)
3. 모바일 및 다른 기기에서의 교차 테스트 (네트워크 접속)
4. 자동 포트 할당 및 브라우저 실행
