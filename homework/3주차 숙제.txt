도메인과 DNS란 무엇인가?

도메인: 
-특정 망(network)의 영역을 구분하기 위한 용어. 가장 좁게는 호스트명(hostname)의 영역 구분 방식 및 체계
-인터넷상의 컴퓨터나 서버를 식별하기 위해 사람이 읽고 기억하기 쉽게 만든 문자형 주소(예: google.com). 
-네트워크 통신의 실제 목적지인 숫자 형태의 IP 주소를 대체하여 사용 편의성을 제공

DNS (Domain Name System): 
-숫자의 연속인 IP 주소를 일일이 외울 수 없기 때문에 쉽게 기억할 수 있는 도메인 네임을 사용하는 것
-사람이 입력한 도메인 이름을 컴퓨터가 이해할 수 있는 IP 주소(예: 192.0.2.1)로 변환해 주는 시스템 
-인터넷의 '전화번호부' 역할을 수행하며, 분산형 데이터베이스를 통해 도메인과 IP 주소의 매핑 정보를 저장하고 클라이언트의 요청에 응답

DNS 서버가 고장나면 웹사이트 접속은 어떻게 될까?
기본적으로 DNS 서버가 마비되면 도메인 이름을 IP 주소로 변환하는 과정이 실패하므로, 도메인을 통한 정상적인 웹사이트 접속이 불가능
사용자의 웹 브라우저가 목적지의 실제 위치를 찾을 수 없기 때문에 'DNS_PROBE_FINISHED_NXDOMAIN' 또는 '서버 IP 주소를 찾을 수 없음' 등의 네트워크 에러를 반환

예외적으로 접속이 가능한 경우

단, DNS 서버 장애 상황에서도 논리적으로 다음의 조건에서는 서버와의 통신 및 웹사이트 접속이 가능

1. IP 주소 직접 입력: 웹 서버의 실제 IP 주소를 알고 있다면, 브라우저 주소창에 도메인 대신 IP 주소를 직접 입력하여 접속할 수 있습니다. 이 경우 DNS 변환 단계를 완전히 생략하므로 DNS 서버의 상태와 무관하게 통신이 성립

2.로컬 캐시(Cache) 정보 활용: 사용자의 운영체제(OS)나 웹 브라우저는 빠른 통신을 위해 최근 방문한 사이트의 도메인-IP 매핑 정보를 로컬 환경에 임시로 저장
DNS 서버가 응답하지 않더라도 해당 기기의 로컬 캐시에 IP 정보가 남아있고 레코드의 유효 시간(TTL)이 만료되지 않았다면 정상적으로 접속 가능
