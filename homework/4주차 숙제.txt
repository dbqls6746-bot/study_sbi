프론트엔드와 백엔드의 역할
+ 회원가입 페이지를 만들 때 어떤 부분이 프론트이고 어떤 부분이 백엔드일까?

프론트엔드-> 사용자가 직접 보고 상호작용하는 화면과 경험 구현
ex)식당 홀, 웨이터

백엔드-> 눈에 보이지 않는 곳에서 데이터 저장, 처리, 보안 담당
ex)식당 주방, 식자재 창고

회원가입 페이지
1. 프론트엔드의 역할 (화면과 사용자 경험)
UI 배치: 이메일 입력 칸, 비밀번호 입력 칸, 그리고 '가입하기' 버튼을 화면에 보기 좋게 생성 및 배치

1차 유효성 검사 : 사용자가 비밀번호를 3자리만 입력하면, 서버로 데이터를 보내기도 전에 화면에 "비밀번호는 8자리 이상이어야 합니다"라는 빨간색 경고 문구를 즉각적으로 보여줌

데이터 전송: 사용자가 '가입하기' 버튼을 누르면, 입력된 정보를 잘 포장해서 백엔드 서버로 전송, 이때 화면에 로딩 스피너(빙글빙글 도는 아이콘)를 보여주어 처리 중임을 알림

결과 안내: 백엔드의 응답에 따라 환영 페이지로 화면을 전환할지, "이미 가입된 이메일입니다"라는 팝업을 띄울지 결정

2. 백엔드의 역할 (데이터 처리와 보안)
데이터 수신 및 2차 검증: 프론트엔드가 보낸 데이터를 받습니다. 누군가 악의적으로 시스템을 우회해 비정상적인 데이터를 보냈을 수 있으므로, 규칙에 맞는지 반드시 한 번 더 꼼꼼하게 검사

중복 확인: 데이터베이스라는 정보 창고를 뒤져서 방금 들어온 이메일이 이미 가입된 적이 있는지 대조

비밀번호 암호화: 보안상 가장 중요한 부분 사용자의 비밀번호를 있는 그대로 저장하지 않고, 알아볼 수 없는 복잡한 난수 문자열로 변환

DB 저장 및 결과 응답: 모든 확인이 끝나면 새 회원의 정보를 DB에 안전하게 기록하고, 프론트엔드에게 "가입 처리 성공"이라는 확인 도장을 찍어 반송


요약: 회원가입 시 프론트엔드는 사용자가 쉽고 편하게 입력할 수 있도록 돕고, 백엔드는 그 정보가 안전하고 정확하게 시스템에 등록되도록 철저히 검증하고 보관
