## 1. 자바 프로그램의 기본 구조와 입출력

* 
**입력 (`Scanner`):** 키보드로부터 통문자열을 쉽게 읽기 위해 사용합니다.

## 2. 변수와 데이터 타입

변수는 값을 저장할 수 있는 메모리 번지이며, 선언된 블록 `{ }` 내부에서만 사용할 수 있습니다. 초기화되지 않은 변수는 사용할 수 없습니다.

**데이터 타입의 종류:**

* 
**정수 타입:** `byte`(1byte), `short`(2byte), `char`(2byte, 유니코드), `int`(4byte), `long`(8byte).


* 
**실수 타입:** `float`(4byte), `double`(8byte). 실수 리터럴은 기본적으로 `double`로 인식되므로 `float` 저장 시 끝에 `f`를 붙여야 합니다.


* 
**논리 타입:** `boolean` (`true`, `false` 저장).


* 
**문자열:** `String` (기본 타입은 아니며, 큰따옴표 `"`로 감싸서 표현).



**타입 변환 코드 활용:**

* 
**자동 타입 변환:** 작은 허용 범위 타입이 큰 허용 범위 타입으로 저장될 때 발생합니다.


* 
**강제 타입 변환 (Casting):** 캐스팅 연산자 `()`를 사용하여 큰 타입을 작은 타입으로 강제 변환합니다.


```java
int intValue = 10;
byte byteValue = (byte) intValue; [cite_start]// 강제 타입 변환 [cite: 4858, 4859]

```


* 
**문자열을 숫자로 변환:** `Integer.parseInt()` 또는 `Double.parseDouble()` 등을 사용합니다.


```java
String str = "300";
int value = Integer.parseInt(str); [cite_start]// 문자열을 int로 변환 [cite: 4975]

```

## 3. 연산자

연산자는 산술, 증감, 비교, 논리, 대입 연산자 등이 있으며 우선순위에 따라 처리됩니다.

**주요 연산자 활용:**

* **증감 연산자 (`++`, `--`):** 피연산자의 값을 1 증가시키거나 감소시킵니다. 변수 앞/뒤 위치에 따라 다른 연산과의 수행 순서가 달라집니다.


* 
**비교 연산자 (`==`, `!=`, `<`, `>`):** 대소를 비교하여 `boolean` 값을 산출합니다. 단, 문자열 비교는 `==` 대신 `equals()` 메소드를 사용해야 합니다.


* 
**논리 연산자 (`&&`, `||`, `!`):** AND(논리곱), OR(논리합), NOT(논리 부정)을 수행합니다.


* **삼항 연산자:** `(조건식) ? [cite_start]값1 : 값2` 형태로, 조건식이 `true`이면 값1을, `false`이면 값2를 산출합니다.


* 
**문자열 결합 (`+`):** `+` 연산 시 피연산자 중 하나가 문자열이면 나머지 피연산자도 문자열로 자동 변환되어 결합됩니다.

## 1. 메모리 구조와 참조 타입

* 
**기본 타입 vs 참조 타입:** 기본 타입(int, double 등)은 변수에 실제 값을 직접 저장하지만, 참조 타입(배열, 클래스, 인터페이스 등)은 힙(Heap) 영역에 생성된 객체의 메모리 번지를 저장하여 참조합니다.


* 
**메모리 사용 영역:** 자바의 메모리는 클래스별 정적 필드와 메소드 코드를 저장하는 '메소드 영역', 객체와 배열이 실제로 생성되는 '힙 영역', 메소드 호출 시 로컬 변수들이 저장되는 'JVM 스택 영역'으로 나뉩니다.


* 
**String 타입과 번지 비교:** 참조 타입 변수 간의 `==` 연산은 실제 데이터가 아닌 메모리 번지가 같은지를 비교합니다. 동일한 `String` 리터럴은 힙 영역의 같은 객체를 공유하지만, `new` 연산자를 사용하면 새로운 객체가 생성되므로 내부 문자열을 비교할 때는 반드시 `.equals()` 메소드를 사용해야 합니다.


* 
**예외 처리:** 참조 변수가 아무런 객체도 참조하지 않을 때 `null` 값을 가지며, 이 상태에서 접근을 시도하면 `NullPointerException`이 발생합니다.

## 2. 데이터의 시스템적 그룹화 (배열과 열거 타입)

* 
**배열의 특징:** 동일한 타입의 데이터를 연속된 메모리 공간에 할당하고 인덱스를 부여하는 자료구조로, 한 번 생성되면 그 길이를 변경할 수 없습니다.


* 
**배열 선언과 생성:** 값 목록을 직접 나열하는 `타입[] 변수 = {값1, 값2};` 방식이나 `new 타입[길이];`를 통해 생성합니다. 2차원 배열은 1차원 배열이 또 다른 1차원 배열의 번지를 참조하는 계단식 구조(`new int[2][3]`)로 이루어집니다.


* 
**향상된 for문:** `for(타입 변수 : 배열)` 구조를 통해 인덱스와 카운터 변수 없이 배열의 전체 항목을 효율적으로 순회할 수 있습니다.


* 
**열거(Enum) 타입:** 요일처럼 한정된 개수의 상수 데이터만 취급해야 할 때 `public enum`으로 묶어 시스템 내 데이터의 무결성을 보장합니다.

## 3. 객체 지향 프로그래밍과 설계도 (클래스)

* 
**클래스와 인스턴스:** 클래스는 객체를 만들어내기 위한 소프트웨어 설계도이며, 이 설계도를 바탕으로 `new` 연산자를 통해 힙 영역에 생성된 실체를 인스턴스(객체)라고 부릅니다.


* 
**객체의 구성:** 객체는 상태 데이터를 저장하는 '필드(Field)', 생성 시 초기화를 수행하는 '생성자(Constructor)', 동작과 기능을 수행하는 '메소드(Method)'로 구성됩니다.


* 
**필드(Field):** 클래스 블록 내부에 선언되며, 객체 생성 시 초기값을 지정하지 않으면 타입에 따라 0, false, null 등의 기본값으로 자동 초기화됩니다.


* 
**생성자(Constructor):** 객체 생성 시 자동 호출되어 필드 초기화를 담당합니다. 외부에서 들어오는 데이터 형태에 대응하기 위해 매개 변수를 다르게 하는 '생성자 오버로딩(Overloading)'이 가능하며, 코드 중복을 최소화하기 위해 생성자 내부에서 `this()`를 호출하여 다른 생성자로 처리를 넘길 수 있습니다.


* 
**메소드(Method):** 외부 데이터를 받는 '매개 변수'와 실행 결과값을 반환하는 '리턴(return)문'으로 구성됩니다. 결과값이 불필요한 경우 `void` 타입을 사용하며, 이때 `return;`을 선언하면 메소드 실행을 강제로 종료합니다. 생성자처럼 매개 변수의 타입이나 개수가 다르면 '메소드 오버로딩'이 가능합니다.

## 4. 자원 레버리지와 메모리 최적화 (인스턴스와 정적 멤버)

* 
**인스턴스 멤버:** 객체마다 독립적으로 유지되어야 하는 데이터는 인스턴스 필드로 선언하며, 객체 내부에서 자기 자신의 멤버에 접근할 때는 `this` 키워드를 사용합니다.


* 
**정적(static) 멤버:** 모든 객체가 공유하는 고정된 데이터나 기능은 `static`으로 선언하여 클래스 로딩 시 메소드 영역에 단 한 번만 할당되도록 레버리지를 만듭니다. 객체를 생성하지 않고도 `클래스.필드` 방식으로 즉시 사용할 수 있습니다.


* 
**싱글톤(Singleton):** 시스템 전체에서 단 하나의 객체만 보장해야 할 때, 생성자를 `private`으로 막고 정적 메소드 `getInstance()`를 통해서만 유일한 객체를 반환하도록 통제하는 코딩 기법입니다.


* 
**상수와 final 필드:** `final`로 선언된 필드는 초기화 후 변경이 불가능한 최종값이 되며, `static final`은 불변의 상수(Constant)를 정의하는 데 사용됩니다.

## 5. 데이터 보호와 시스템 격리 (패키지 및 캡슐화)

* **패키지(Package)와 import:** 클래스를 관리하는 폴더 역할을 하며 고유한 식별자 체계를 만듭니다. 다른 패키지의 클래스를 가져올 때는 `import`문으로 컴파일러에 명시해야 합니다.


* 
**접근 제한자(Access Modifier):** `public`(전체 공개), `protected`(같은 패키지 및 상속된 자식), `default`(같은 패키지 내), `private`(클래스 내부만 허용)을 통해 외부의 무분별한 접근을 차단합니다.


* 
**Getter & Setter:** 데이터의 무결성을 유지하기 위해 필드는 `private`으로 차단하고, 검증 로직이 포함된 `Setter` 메소드로 값을 안전하게 수정하며 `Getter` 메소드를 통해 값을 반환하는 캡슐화 원칙을 적용합니다.
